> 版本：v1.0 | 日期：2026-06-11

## 编码习惯

1. **所有文件必须有版本头** — `@version` + 修订记录表。代码文件在开头，Markdown 在末尾。
2. **先定计划再动手** — 要求先出设计文档/计划，审阅通过后再实施。
3. **逐个修复、逐个验证** — 反对一次性大批量修改（"一次性生成的内容太多了"）。
4. **Bug 表格追踪** — 所有 Bug 用表格记录（编号、描述、时间、优先级、状态、修复方案）。
5. **文档与代码同步** — 修改实现必须同步更新设计文档。
6. **版本号递增** — 每次修改文件要更新 `@version` 并追加修订记录。

## 工作环境

7. **CMD 优先** — npm 命令在 CMD 中执行（PowerShell 不支持 npm）。
8. **ELECTRON_RUN_AS_NODE 已知问题** — Windows 环境变量 `ELECTRON_RUN_AS_NODE=1` 会导致 Electron 以 Node 模式运行。启动前需 `set ELECTRON_RUN_AS_NODE=`。
9. **端口 5173 占用** — Vite 默认端口可能被占用，需 `taskkill` 释放。

## UI 偏好

10. **Win+V 风格** — 偏好 Windows 系统剪贴板的小巧紧凑风格（~360×450），不喜大窗口。
11. **置顶在列表顶部** — 置顶不应独立 Tab，应在历史列表最上，用淡红底色区分。
12. **收藏用浅黄底色** — 置顶=淡红、收藏=浅黄，快速区分。
13. **图标始终显示** — 激活=彩色、未激活=灰色，不用条件渲染隐藏。
14. **中英切换** — 需要 i18n，但不着急（P1），功能稳定后再接入。

## 反馈风格

15. **截图标注问题** — 用红框圈出问题区域，清晰直接。
16. **"你可以debug查看一下"** — 鼓励主动排查，不满足于表面修复。
17. **"不要急于修复，基于我提出的内容先修改好计划"** — 强调计划先行。
